# 🤖 ML Bot — Advanced RAG Chatbot

A production-grade **Retrieval-Augmented Generation (RAG)** chatbot that answers questions about a Machine Learning textbook. The system implements and benchmarks three chunking strategies, three embedding models, and two retrieval methods before serving cited answers through a Streamlit UI.

<br>

## Architecture

```
PDF
 │
 ▼
open_and_read_pdf()          ← chunking.py
 │
 ├── Fixed-size chunks
 ├── Recursive chunks  ──────────────────────┐
 └── Hybrid chunks  (production default) ───►│
                                             │
                                             ▼
                             generate_embeddings_for_all()   ← embedding.py
                             (mpnet · bge · minilm)
                                             │
                                             ▼
                             build_and_save_indexes()
                             (6 × FAISS IndexFlatL2)
                                             │
                       ┌─────────────────────┘
                       │   At query time
                       ▼
               mmr_search()          diversity-aware FAISS retrieval
                       │
                       ▼
               rerank_documents()    CrossEncoder (bge-reranker-base)
                       │
                       ▼
               hallucination guard   avg rerank score < threshold → refuse
                       │
                       ▼
               TinyLlama 1.1B (4-bit NF4)  → cited answer
                       │
                       ▼
               Streamlit UI          app.py
```

## Quickstart

### 1 — Clone & install

```bash
git clone https://github.com/harikharasudhan10/advanced-rag-chatbot.git
cd advanced-rag-chatbot

pip install -r requirements.txt

# PyTorch with CUDA 12.1 (adjust for your driver)
pip install torch torchvision torchaudio \
    --index-url https://download.pytorch.org/whl/cu121
```

### 2 — Add your PDF

```bash
cp "/path/to/machine learning book.pdf" .
```

### 3 — Build indexes (run once)

```bash
python build_index.py --pdf "machine learning book.pdf"
```

This generates the `.pkl` and `.bin` files in the current directory.

```
== Step 1 / 4  Reading PDF …
== Step 2 / 4  Chunking …
== Step 3 / 4  Generating embeddings …
== Step 4 / 4  Building FAISS indexes …
✅  All artifacts generated in 142.3s
```

### 4 — Launch the app

```bash
streamlit run app.py
```

Open `http://localhost:8501` in your browser.

## Deployment on Google Colab

Add this cell **after** all setup cells in your notebook:

```python
import subprocess, re, time

# Kill any old processes
subprocess.run(["pkill", "-f", "streamlit"], capture_output=True)
subprocess.run(["pkill", "-f", "cloudflared"], capture_output=True)

# Download Cloudflare tunnel binary
subprocess.run([
    "wget", "-q",
    "https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64",
    "-O", "cloudflared"
])
subprocess.run(["chmod", "+x", "cloudflared"])

# Start Streamlit
subprocess.Popen([
    "streamlit", "run", "app.py",
    "--server.port", "8501",
    "--server.address", "0.0.0.0",
])
time.sleep(10)

# Open public tunnel
tunnel = subprocess.Popen(
    ["./cloudflared", "tunnel", "--url", "http://localhost:8501"],
    stdout=subprocess.PIPE, stderr=subprocess.STDOUT, text=True,
)
time.sleep(15)

for line in iter(tunnel.stdout.readline, ""):
    match = re.search(r"https://[a-zA-Z0-9\-]+\.trycloudflare\.com", line)
    if match:
        print("\n✅ Public URL:", match.group(0))
        break
```

## Module Reference

### `chunking.py`

| Function | Description |
|---|---|
| `open_and_read_pdf(pdf_path)` | Extract per-page text with stats |
| `chunk_pdf_pages(pages)` | Fixed-size word-boundary chunks |
| `recursive_chunk_pdf_pages(pages)` | Hierarchy-aware recursive split |
| `semantic_chunk_pdf_pages(pages)` | Split at semantic similarity drops |
| `hybrid_chunk_pdf_pages(pages)` | Recursive base + semantic merge ← **default** |
| `visualize_chunks(chunks)` | Pretty-print a random sample |

### `embedding.py`

| Function | Description |
|---|---|
| `load_embedding_models(device)` | Load mpnet · bge · minilm |
| `generate_embeddings_for_all(chunk_sets, models)` | Encode all chunk × model combos |
| `save_embeddings(store)` / `load_embeddings(path)` | Pickle I/O |
| `save_chunk_sets(sets)` / `load_chunk_sets(path)` | Pickle I/O |
| `build_and_save_indexes(store)` | Build + save all FAISS indexes |
| `load_faiss_index(chunk_type, model_name)` | Load one index from disk |
| `mmr_search(...)` | Diversity-aware FAISS retrieval |
| `load_reranker(model_name)` | Load CrossEncoder |
| `rerank_documents(query, chunks, reranker)` | Score and sort by relevance |

### `build_index.py`

```
python build_index.py [--pdf PATH] [--device cuda|cpu]
                      [--max-chunk-size N] [--min-chunk-size N]
```

### `app.py`

Streamlit application — run with `streamlit run app.py`.

## Chunking Strategies

| Strategy | Key Idea | Use Case |
|---|---|---|
| **Fixed-size** | Word-boundary windows of N chars | Baseline |
| **Recursive** | Paragraph → line → sentence → word | Precise atomic chunks |
| **Semantic** | New chunk when cosine similarity drops | Topic-aware splitting |
| **Hybrid** ★ | Recursive base + semantic merge | Best relevance + context |

★ Production default.

<br>

## Embedding Models

| Key | Model | Dim |
|---|---|---|
| `mpnet` | `all-mpnet-base-v2` | 768 |
| `bge` | `BAAI/bge-small-en` | 384 |
| `minilm` | `all-MiniLM-L6-v2` | 384 |

All embeddings are **L2-normalised** before indexing.

## Retrieval Pipeline

```
Query
  │
  ▼  encode with MiniLM
Query Embedding
  │
  ▼  FAISS IndexFlatL2 → top-20 candidates
MMR (λ = 0.7)
  │  balance relevance vs. diversity
  ▼
Top-20 diverse candidates
  │
  ▼  BAAI/bge-reranker-base CrossEncoder
Reranked top-5
  │
  ▼  avg rerank score < 0.05 → refuse
Hallucination Guard
  │
  ▼
TinyLlama 1.1B (4-bit NF4)
  │  max_new_tokens=150, temperature=0.0
  ▼
Cited Answer  [1] [2] …
```

<br>

## Evaluation Summary

Query used: *"What is machine learning about?"*

| Chunk Type | Model | Top-1 L2 ↓ | Notes |
|---|---|---|---|
| Recursive | **BGE** | **0.24** | Best single-result precision |
| Hybrid | **BGE** | **0.24** | Best context richness |
| Recursive | MiniLM | 0.46 | Strong second |
| Hybrid | MPNet | 0.63 | Broader merges dilute direct match |

MMR consistently outperformed pure similarity search by surfacing distinct subtopics (supervised learning, reinforcement learning) instead of near-duplicate paragraphs.

## Tech Stack

| Layer | Library / Model |
|---|---|
| PDF parsing | PyMuPDF (`fitz`) |
| Sentence tokenisation | NLTK |
| Embedding | SentenceTransformers |
| Vector index | FAISS `IndexFlatL2` |

| UI | Streamlit |

## Configuration

All defaults live at the top of `app.py`:

```python
LLM_MODEL_ID        = "TinyLlama/TinyLlama-1.1B-Chat-v1.0"
DEFAULT_EMBED_MODEL = "minilm"
DEFAULT_CHUNK_TYPE  = "hybrid"
DEFAULT_K           = 5       # final chunks sent to LLM
DEFAULT_LAMBDA      = 0.7     # MMR: 1.0 = relevance, 0.0 = diversity
DEFAULT_FETCH_K     = 20      # initial FAISS candidate pool
AVG_THRESHOLD       = 0.05    # hallucination guard cutoff
```
